# udep-wow

Node.js / Express API — bootstrapped by UDAP.

## Project structure

```
├── server.js                  # Entry point (starts the HTTP server)
├── src/
│   ├── app.js                 # Express app factory (middleware + routes)
│   ├── routes/
│   │   ├── index.js           # Route registrar
│   │   ├── health.js          # GET /health
│   │   └── api.js             # GET /api/info — add your routes here
│   ├── middleware/
│   │   ├── errorHandler.js
│   │   └── requestLogger.js
│   ├── config/
│   │   └── database.js    # DB connection pool
├── public/                    # Static files (optional)
├── Dockerfile
├── .env.example
└── .github/workflows/         # CI/CD (generated by UDAP)
```

## Local development

```bash
cp .env.example .env   # fill in your values
npm install
npm run dev            # starts with --watch (hot-reload)
```

## API endpoints

| Method | Path        | Description          |
|--------|-------------|----------------------|
| GET    | /health     | Health check         |
| GET    | /api/info   | App info + DB status |


## Database

This project uses **Postgres**. Set the connection variables in `.env` before running.
## Deployment

Push to `main` — GitHub Actions will provision infrastructure and deploy automatically.
Check `.github/workflows/deploy.yml` for the pipeline definition.
